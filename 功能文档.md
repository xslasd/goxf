`goxf` 是一个基于 Go 语言的高性能、企业级微服务开发框架。它秉承着**高内聚、低耦合**的设计理念，将微服务开发生命周期中常用的基础组件进行模块化组装，开箱即用，极大地提高了业务侧的开发效率。

---

## 1. 核心运行容器 (`application`, `goxf.go`)

- **服务编排体系**：提供全局统一的 `Service` 容器对业务环境进行托管。支持通过 `goxf.NewService()` 链式启动各个下游协议组件。
- **配置化组件按需加载**：日志、配置解析、监控分析等所有内部模块依赖均高度抽象化，并依赖全局上下文相互编排。
- **生命周期 Hook 机制**：内建基于闭包回调的四阶段 Hook（`Stage_BeforeLoadConfig`、`Stage_BeforeRun`、`Stage_BeforeStop`、`Stage_AfterStop`），允许用户在服务启停阶段自由注入额外的拦截、清理或初始化逻辑。
- **优雅的平滑启停**：深度集成系统级退出信号（SIGQUIT, SIGTERM, SIGINT）嗅探处理，确保所有的数据库连接池、分布式追踪数据、网络 HTTP Server 等能在进程完全关闭前“零丢失”地安全退出。

## 2. 工程化配置管理 (`conf`)

- **多数据源及格式解析**：默认支持 `YAML`、`JSON`、`TOML` 等常见配置文件协议解析。
  - **自定义 Unmarshal**：在实例化配置源（如 `NewSourceConf`）时，框架允许且**优先应用业务方自定义的 `Unmarshal` 处理器**。开发者能够完全接管文件字节流到结构体的解析过程（例如：在解析前统一进行特殊环境变量替换或自定义语法解包），兼顾统一性与极高的定制自由度。
- **配置本地化覆盖（`.local` 机制）**：
  - 当框架加载主配置文件（如 `config.yaml` 或 `config.toml`）时，会自动嗅探同级目录下是否存在带有 `.local` 后缀的私有配置文件（如 `config.local.yaml`）。
  - 若存在，框架会自动读取并执行**深度合并（Merge）**，优先使用本地配置项覆盖主配置。这非常有利于团队协作：开发者可以在本地创建 `.local` 文件来调试数据库和测试环境，而不必担心误提交污染 Git 仓库的主配置。
- **配置热更新及动态订阅**：
  - 启动时可以通过 `-watch` 形参开启配置监听守护进程。框架后台具备多路复用能力，主配置与 `.local` 配置的任何动态修改均能被合并触发。
  - 配置被动态拉取并覆写时，支持通过回调精准推回给对应注册模块（例如，**无需重启服务器即可在运行时动态切级修改线上的 Log 输出门槛等**）。
- **安全性与加密（`system.enc`）**：
  - **基于 SM4 国密算法**：对敏感配置文件支持加解密存储，适配企业级安全合规要求。
  - **一键配置编译**：当携带命令行参数 `--crypt-conf` 启动框架时，将进入交互式密码验证模式：
    - 若读取到的是明文配置（包含本地合并后的最新状态），框架会将其使用密码进行高强度加密，生成防篡改的 `system.enc` 密文文件，并输出警告提示开发者及时删除明文。
  - **运行时透明解密**：应用在生产环境启动时，只需通过安全手段注入密码（如 `conf.SetPassword()` 或环境变量），框架会自动探测并直接读取 `system.enc` 进行内存透明解密。
  - 为了极高的生产安全性，一旦系统判断当前挂载的是 `system.enc` 加密文件，底层文件 Watch 监听机制会被强制禁用，杜绝非法热替换。

## 3. 高性能服务组网 (`server`, `client`)

- **多协议标准化 Server 设计**：抽象出一套无侵入的 `server.Server` 标准，包含预留 HTTP（`sgin/gin`）、RPC、WebSocket、以及可能延展的自定义私有协议，实现统一管控并行执行。
- **业务中间件自动注入**：针对 HTTP 网关，集成标准且全面的跨域 (`CORS`) 及其鉴权(`Auth/JWT`) 生命周期守护网卡收敛。
- **健壮的外部 Client 生态**：内置原生级别的常用存储介质交互适配器：
  - `credis`：标准 Redis 分布式交互客户端。
  - `cetcd`：处理分布式配置、节点锁、注册订阅的 Etcd 工具集。
  - `carangodb`：图数据库 ArangoDB 交互等。

---

## 4. 强大的统一日志台 (`log`)

- **无极性能拓展**：底层深度封装了 Uber `zap` 标准引擎。
- **异步日志与归档切分**：可通过配置轻松实现文件日志落盘缓存刷新（可异步 `bws` 处理高并发），与标准自动切割归档策略（大小、时间）。
- **组件日志隔离适配**：能无缝适配/劫持 gRPC、XORM 引擎、Jaeger 等框架的私有日志接口并将收敛标准统一到 goxf 内。

---

## 5. 多维度服务治理扩展

- **JWT / 签名与鉴权体系 (`auth`)**：实现企业级权限安全签名管控 `JWTCertifier` 及令牌全生命周期处理机制。
- **微服务可观测性分析：**
    - **`metric` (Prometheus 等)**：无缝嵌入指标暴露器分析应用健康态。
    - **`tracer` (Jaeger 链路追踪)**：为分布式进程网提供标准、轻量的 `OpenTracing` 链路信息。
- **强类型后台任务引擎 (`job`)**：提供生产级别、高度抽象的后台任务托管方案：
    - **`job/cron` (定时任务)**：基于配置化调度的标准 Cron 任务引擎，支持秒级调度。
    - **`job/queue` (异步队列 & 延迟任务)**：内置通用的泛型句柄驱动（`Worker[T]`），实现消息的强类型分发与无缝断言。
        - **底层 Broker 驱动抽象**：提供 `MemoryBroker` (纯内容极管、基于时间轮优化的高性能 O(1) 协程分配) 及 `credis.QueueBroker` (持久化的高稳定 Redis 分发机制)。
        - **延迟任务 (Delayed Jobs)**：提供开箱即用的延迟投递功能 `EnqueueAfter`。在 Redis 下通过 ZSET+Lua 高可靠保障无丢失，利用极简接口优雅应对“订单超时判定/延迟回调”场景。
    - 均已标准收归框架生命周期守护，默认注入所有 Panic 恢复体系与底层 Prometheus 监控（`ServerHandleHistogram` / `ClientHandleHistogram`）。
- **微服务状态监控 (`registry`)**：预留灵活规范的服务注册与发现（Registry / Selector），以极速拓展在 Service Mesh 或纯微服务内环境通信的协调操作。

---

## 6. 标准化基础构建块

- **全球化 i18n (`i18n`)**：提供标准且灵活的多语言本地化翻译管理映射器。
- **错误码收敛字典 (`ecode`)**：构建带有业务语素解析的全项目定制化 `common_ecode` 表以及灵活透传框架，支持无损附带错误链定位（Stacktrace）。
- **便捷开发者工具 (`util`)**：包含格式化 `xfmt` 输出等内部精简直观的方法增强日常开发可读性。
